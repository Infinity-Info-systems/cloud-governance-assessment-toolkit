# Assessment Plan

This plan turns the cloud governance toolkit into a repeatable assessment workflow.

## Purpose

Use this plan to collect evidence, score maturity, and produce a remediation roadmap for cloud governance, SRE, DR, FinOps, and security controls.

## Assessment Phases

### 1. Scope

- define in-scope business units and platforms
- identify the assessment owners
- confirm time window and evidence sources

### 2. Evidence Collection

- questionnaires
- checklists
- interviews
- dashboards and reports
- policy and control documents

### 3. Scoring

- score each domain against the toolkit model
- record the current state and target state
- capture confidence level and gaps

### 4. Findings Review

- validate findings with stakeholders
- rank risks by impact and urgency
- identify quick wins and structural issues

### 5. Remediation Planning

- assign owners
- sequence actions by dependency
- define target dates
- map actions into a roadmap

## Example Assessment Output

| Domain | Score | Gap | Priority |
| --- | --- | --- | --- |
| Cloud Governance | 3/5 | Policy execution consistency | High |
| SRE Maturity | 2/5 | Alert quality and service ownership | High |
| DR Readiness | 3/5 | Test frequency and evidence | Medium |
| FinOps Readiness | 2/5 | Tagging and commitment governance | High |
| Security Compliance | 4/5 | Exception handling discipline | Medium |

## Recommended Actions

- use one questionnaire set across all domains
- produce one executive summary for leadership
- keep the remediation roadmap separate from the findings log
- revisit scores after remediation cycles
